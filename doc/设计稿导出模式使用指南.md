# 设计稿导出模式 —— 使用指南

> 适用工具：html.to.design（Figma 插件）  
> 适用场景：将 H5 原型页面转为 Figma 设计稿，确保与小程序实际效果一致

---

## 一、导出前的两步准备

### 步骤 1：设置移动端视口宽度

打开 Chrome DevTools（F12），切换到设备模拟模式，选择合适的设备尺寸：

| 设备 | 宽度 | 高度 | 推荐场景 |
|:---|:---|:---|:---|
| **iPhone 14 / 15** | **390px** | **844px** | ✅ **推荐，当前主流基准** |
| iPhone 14 / 15 Pro | 393px | 852px | 也可以，差异极小 |
| iPhone 13 mini | 375px | 812px | 旧基准，逐渐淘汰 |

**推荐使用 390×844**，这是目前设计行业最主流的 iPhone 移动端设计基准尺寸。

> **操作方法**：F12 → 点击左上角"设备切换"图标 → 顶部选择"iPhone 14" 或手动输入 390×844

### 步骤 2：启用小程序模拟模式

在页面 URL 后加 `?design=mp` 参数：

```
index.html?design=mp        → 首页
customers.html?design=mp    → 名单页
follow.html?design=mp       → 跟进页
detail.html?design=mp       → 详情页
```

页面顶部会自动出现模拟的微信小程序原生导航栏（状态栏 + 标题栏 + 胶囊按钮）。

## 二、逐页导出操作

以下是每个页面的完整导出 URL（假设本地文件路径）：

| 页面 | URL |
|:---|:---|
| 首页 | `file:///...bank-mobile/index.html?design=mp` |
| 名单页 | `file:///...bank-mobile/customers.html?design=mp` |
| 跟进页 | `file:///...bank-mobile/follow.html?design=mp` |
| 详情页 | `file:///...bank-mobile/detail.html?design=mp` |

**每个页面的操作流程**：

1. F12 打开 DevTools → 设备模式 → 选择 iPhone 14（390×844）
2. 地址栏输入页面 URL + `?design=mp` → 回车
3. 确认顶部出现模拟导航栏（有返回箭头、标题、右侧胶囊按钮）
4. 在 Figma 中启动 html.to.design 插件 → 粘贴当前页面 URL → 导入
5. 导入后在 Figma 中得到的就是小程序内的真实效果

## 四、设计模式的工作原理

当 URL 包含 `?design=mp` 参数时，`js/design-mode.js` 脚本会自动执行以下操作：

1. **注入模拟导航栏** —— 在页面顶部插入仿真的微信原生导航栏 DOM（状态栏 44px + 导航栏 44px = 88px）
2. **激活适配 CSS** —— 在 `<html>` 上添加 `.in-miniprogram` class，触发 `css/style.css` 中的适配规则
3. **页面级调整** —— Header 去除安全区 padding、隐藏返回按钮、隐藏头像等

**不影响正常使用**：不加参数直接打开页面，效果与普通 H5 完全一致。

## 五、手动控制（Console 命令）

除了 URL 参数，也可以在浏览器 Console 中手动切换：

```javascript
// 启用小程序模拟模式
enableDesignMode()

// 退出小程序模拟模式
disableDesignMode()
```

## 六、FAQ

**Q：为什么要用 390px 而不是 375px？**  
A：375px 是 iPhone X/11 时代的基准，现在 iPhone 14/15 系列的逻辑宽度是 390px，是设计行业当前的主流基准。Figma 中的 iPhone 14 Frame 也是 390×844。用 390px 导出的设计稿在 Figma 中可以直接放入标准 Frame。

**Q：html.to.design 需要访问线上地址吗？本地文件能用吗？**  
A：html.to.design 支持输入 URL。本地文件（`file:///` 协议）可能需要先起一个本地 HTTP 服务器。可以用 `npx serve bank-mobile` 快速启动，然后用 `http://localhost:3000/index.html?design=mp` 访问。

**Q：导出的设计稿需要做什么后处理？**  
A：导入 Figma 后建议检查：字体是否正确加载、SVG 图标是否完整、颜色值是否准确。模拟导航栏可以在 Figma 中替换为官方的小程序 UI Kit 组件以获得更精确的效果。
